## Recommended Sync Strategy

**Do NOT attempt `git merge upstream/main`.** The divergence is too deep (39 vs 100+ commits, massive structural changes on both sides, dirty tree). A merge would produce dozens of conflicts in the highest-churn files (HomeScreen, ChatPanel, ChatViewModel, GalleryNavGraph, ModelManagerViewModel) and would be nearly impossible to review.

**Strategy: file-by-file / module-by-module adoption in themed batches**, committing checkpoints after each batch. This is the continuation of your current approach.

The workflow per batch:
1. `git stash` or commit current dirty state as a WIP checkpoint
2. `git diff upstream/main HEAD -- <file>` to see what upstream changed
3. Adopt upstream's version of the file, then re-apply fork-specific patches on top
4. Build-verify after each batch (`./gradlew assembleDebug` via CI or local)
5. Commit with a descriptive message
