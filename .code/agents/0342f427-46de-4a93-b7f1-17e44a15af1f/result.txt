# Plan

The safest path to upstream parity is a phased catch-up, not a merge-first sync. This fork is carrying two intentional divergences at once: a large local HTTP bridge centered on [LlmHttpService.kt](/home/zyach/projects/gallery/Android/src/app/src/main/java/com/google/ai/edge/gallery/service/LlmHttpService.kt#L45) and an in-progress removal of the chat-history subsystem across nav/DI/chat/home. Treat those as the stabilization baseline, then pull upstream in small feature bands with GitHub Actions as the primary compile gate after each band.

## Requirements
- Preserve local-only loopback behavior on `127.0.0.1` and keep bridge settings stable via [LlmHttpPrefs.kt](/home/zyach/projects/gallery/Android/src/app/src/main/java/com/google/ai/edge/gallery/data/LlmHttpPrefs.kt#L14) and [SettingsDialog.kt](/home/zyach/projects/gallery/Android/src/app/src/main/java/com/google/ai/edge/gallery/ui/home/SettingsDialog.kt#L96).
- Finish removing chat-history code from the critical path before broad upstream sync work.
- Prefer small themed batches over a giant merge because the tree is already dirty and the main hotspots overlap with current work.
- Use GitHub Actions APK builds as the default validation path via [android-apk.yml](/home/zyach/projects/gallery/.github/workflows/android-apk.yml#L1).

## Scope
- In: Android app parity work, upstream feature adoption, chat-history unwind, loopback bridge hardening, CI-driven validation.
- Out: rewriting the HTTP bridge to match upstream architecture, resurrecting chat history, large repo-wide cleanup unrelated to upstream parity.

## Recommended sync strategy
- Do not start with `git merge upstream/main` on the current dirty tree.
- First checkpoint the current in-flight work on a dedicated branch, then continue with themed adoption branches from that checkpoint.
- Use this order:
  1. Finish local refactors that reduce conflict volume.
  2. Cherry-pick low-conflict upstream commits directly when they touch only a couple files.
  3. For medium/high-conflict areas, do file-by-file adoption against `upstream/main` with manual reconciliation.
  4. Leave a full merge attempt until the fork is functionally close and chat-history removal is complete.
- Practical split:
  - Cherry-pick candidates: `bfd6a63` max-token/config-dialog polish, `14c3089` audio-only message send, `1bf07d6` imported-model runtime default if still applicable.
  - Manual adoption candidates: `15a7e8e` thinking config, `9a5e193` chat-panel thinking UX, `62bcf0e` system prompt editing, internal model/runtime cleanup bands.
  - Avoid full-merge-first on files with local bridge logic or current deletions.
