## Sync Strategy

**Do NOT attempt `git merge upstream/main`** from the current dirty state. The tree has 39 modified/deleted files, the history diverged significantly, and a merge would produce hundreds of conflicts across files that are being simultaneously modified in-flight.

**Recommended approach: file-by-file, phase-by-phase adoption:**

1. **For fork-only files** (`LlmHttpService.kt`, `LlmHttpPrefs.kt`): never merge from upstream (they don't exist there). Update manually in response to upstream API changes.

2. **For purely upstream files with no fork changes** (new `runtime/` package, `ui/benchmark/`, `GlobalModelManager.kt`, new `ui/common/chat/Message*.kt` files): use `git show upstream/main:<path> > <local-path>` to copy verbatim.

3. **For hotspot files** (nav graph, `LlmChatViewModel`, `ChatPanel`, home screen): use `git diff HEAD upstream/main -- <file>` to understand the full delta, then apply changes manually to the fork version, resolving conflicts by hand.

4. **For `build.gradle.kts` and `libs.versions.toml`**: use upstream as base, re-add NanoHTTPD dependency block.

5. **Cherry-pick** is appropriate for isolated, low-risk upstream commits (allowlist updates, string additions, `bfd6a63` config dialog) where the file has no fork changes.
